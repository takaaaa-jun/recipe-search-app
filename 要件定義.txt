要件定義書 (ドラフト)
プロジェクト概要
Blockly を使用して、食材と調理アクションを組み合わせることでレシピ手順を作成できる Web アプリケーション。 データベースに定義された「食材」と「アクション」の制約に基づき、正しい組み合わせのみを許可する。

機能要件
1. フロントエンド (レシピ作成画面)
ワークスペース機能
ブロックをドラッグ＆ドロップで配置できること。
ズーム、ゴミ箱、スクロールなどの基本操作ができること。
ツールボックス (パレット)
「食材」と「アクション」がカテゴリ分けされて表示されること。
データベースのマスタデータに基づいて動的に項目が生成されること。
ブロック定義
食材ブロック: 食材名を表示。
アクションブロック: 調理法（切る、焼くなど）を表示。食材ブロックを入力として受け取る。
バリデーション (制約チェック)
特定の食材に対して、DBで定義された許可されたアクションしか接続できないように制御すること（例: 「トマト」には「切る」はOKだが「こねる」はNG、など）。
レシピ生成・保存
作成したブロックの組み合わせを JSON 形式などのデータとして出力できること。
作成したレシピを保存できること（バックエンド連携）。
2. バックエンド (API)
マスタデータ取得 API
食材一覧、アクション一覧、およびその組み合わせルールを取得するエンドポイント。
レシピ保存/取得 API
ユーザーが作成したレシピデータの保存と読み込み。
3. データ構造 (想定)
テーブル: ingredients_actions (または類似のマスタテーブル)
id: ID
category_medium: カテゴリ (野菜, 肉, 魚など)
food_name: 食材名
action: アクション名
非機能要件
ユーザビリティ: 直感的に操作できること。
拡張性: 新しい食材やアクションをDBに追加するだけで、フロントエンドの変更なしに反映されること。
